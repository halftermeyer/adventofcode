## Evolution of Approach

### 2021-2022: Pioneer Era
Pure Cypher with APOC and GDS. Every puzzle solved by modeling the problem as a graph, then querying it. Heavy use of `apoc.periodic.commit()` for iteration, `apoc.nodes.link()` for linked lists, `gds.shortestPath.dijkstra` for pathfinding. **50 stars** in 2022 proved Cypher could handle any AoC puzzle.

### 2023-2024: Hybrid Era
Python notebooks orchestrating Neo4j queries. Python handles parsing and orchestration; Neo4j handles graph operations, pattern matching, and algorithms. Best of both worlds.

### 2025: CYPHER 25 Revolution
Pure CYPHER 25 — the new language features (`LET`, `NEXT`, `WHEN`, `allReduce`, `REPEATABLE ELEMENTS`, `IN TRANSACTIONS`) make Cypher a complete computation language. `reduce()` replaces Python loops entirely.

### 2015-2018: LLM Retroactive Solutions
Generated by Claude Opus using a Cypher skill trained on all prior years. All answers verified via AoC submission API with **zero wrong answers** on verification. Demonstrates that the skill+cookbook approach produces correct, idiomatic Cypher.

## Key Cypher Techniques Across All Years

| Technique | Years | Description |
|-----------|-------|-------------|
| `reduce()` | 2015-2018, 2025 | Turing-complete fold — implements VMs, minimax, Gaussian elimination |
| `head([v IN [expr] \| ...])` | 2018, 2025 | Let-binding inside reduce (no LET available there) |
| `apoc.periodic.commit()` | 2021-2022 | Iterate-until-done (replaced by IN TRANSACTIONS in 2025) |
| `apoc.nodes.link()` | 2021-2022 | Build linked lists |
| `gds.shortestPath.dijkstra` | 2022, 2023 | Weighted pathfinding |
| `gds.wcc.stream()` | 2022, 2023, 2025 | Connected component detection |
| `allReduce` | 2025 | Stateful path pruning during traversal |
| `REPEATABLE ELEMENTS` | 2025 | Allow revisiting nodes in paths |
| `SHORTEST 1` with predicate | 2025 | Shortest path with complex accumulated constraint |
| Bitwise via bit decomposition | 2018, 2017 | AND/OR/XOR without native bitwise ops |

## Cypher is Turing-Complete

We formally proved it. The [`paper/`](paper/) directory contains a LaTeX article demonstrating that `reduce()`, `CASE`, and list comprehensions can simulate any 2-counter machine (Minsky 1967). Both a full single-column version and a dense 2-column version are included.
